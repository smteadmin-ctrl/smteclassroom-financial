📖 บทที่ 1: บทนำและความรู้พื้นฐาน

ระบบ Classroom Finance คืออะไร

Classroom Finance 5.0 เป็นระบบจัดการการเงินห้องเรียนที่ได้รับการออกแบบเพื่อช่วยเหรัญญิกหรือผู้ดูแลห้องเรียน ในการจัดการรายรับรายจ่ายของห้องเรียนในแบบที่ง่ายและมีประสิทธิภาพ ระบบนี้ใช้เทคโนโลยีสมัยใหม่เพื่อให้สามารถ ติดตามการเก็บเงินจากนักเรียน บันทึกรายรับรายจ่าย และตรวจสลิปการโอนเงินได้อย่างอัตโนมัติ

ส่วนประกอบของระบบ

🌐 เว็บแอปพลิเคชัน สำหรับเหรัญญิกหรือผู้บริหารห้องเรียน ใช้ Chrome, Firefox, Safari บนคอมพิวเตอร์ทุกเครื่อง
📱 LINE Bot สำหรับนักเรียน ช่วยให้นักเรียนดูสถานะการชำระเงิน โอนเงิน และส่งสลิป ผ่าน LINE Official Account
☁️ ฐานข้อมูล Supabase เก็บข้อมูลนักเรียน กำหนดการ ธุรกรรม และรูปสลิป ซึ่งสามารถเข้าถึงจากทุกที่
📊 การวิเคราะห์ข้อมูล แสดงกราฟและสถิติการเก็บเงิน ยอดคงเหลือ และแนวโน้มการเงินห้องเรียน

ประโยชน์ของระบบ

  • บันทึกอัตโนมัติ: ข้อมูลการเก็บเงินจาก LINE บันทึกลงระบบโดยอัตโนมัติ
  • ตรวจสลิปอัตโนมัติ: ระบบสามารถอ่านข้อมูลจากรูปสลิปและตรวจสอบความถูกต้อง
  • หลายช่องทางชำระเงิน: รองรับการโอนธนาคาร TrueMoney และเงินสด
  • ดูแลง่าย: ไม่ต้องแก้โค้ด สามารถปรับตั้งค่าได้ทั้งหมดจากหน้า Settings
  • ปลอดภัย: ข้อมูลเก็บไว้ในระบบคลาวด์ที่มีการเข้ารหัส
✓ ความรู้ที่ต้องการ: ไม่จำเป็นต้องมีพื้นฐานเทคนิคใด ๆ เพียงแค่มีความสนใจที่จะเรียนรู้และใช้งานเท่านั้น

📖 บทที่ 2: การติดตั้งและเตรียมระบบ

ความต้องการทางด้านเทคนิค

รายการ ข้อกำหนด หมายเหตุ
Node.js เวอร์ชัน 18 หรือสูงกว่า (แนะนำ LTS) ดาวน์โหลดจาก nodejs.org
npm มาพร้อมกับ Node.js ใช้สำหรับติดตั้ง packages
Git แนะนำเวอร์ชันล่าสุด ใช้โคลนโครงการ (ถ้าเป็นครั้งแรก)
เบราว์เซอร์ Chrome, Firefox, Safari ใหม่ ๆ ใช้เข้าเว็บแอปพลิเคชัน
บัญชี Supabase สมัครฟรีที่ supabase.com ใช้เป็นฐานข้อมูล

ขั้นตอนที่ 1: ตรวจสอบ Node.js

1. เปิด Terminal (macOS/Linux) หรือ Command Prompt (Windows)

macOS: ค้นหา "Terminal" ในเมนู Spotlight (Cmd + Space)
Windows: ค้นหา "Command Prompt" หรือ "PowerShell"

2. พิมพ์คำสั่งนี้: 
node --version

ถ้าเห็นเลขเวอร์ชัน เช่น v18.0.0 หรือสูงกว่า แสดงว่า Node.js ติดตั้งแล้ว ✓
ถ้าเห็นข้อความ "command not found" ให้ดาวน์โหลดและติดตั้งจาก nodejs.org

3. ตรวจสอบ npm 
npm --version

ควรเห็นเลขเวอร์ชัน เช่น v9.0.0 หรือสูงกว่า

ขั้นตอนที่ 2: โคลนโครงการ (คัดลอกโค้ดจาก GitHub)

1. สร้างโฟลเดอร์สำหรับเก็บโครงการ 
mkdir ~/classroom-finance
cd ~/classroom-finance
2. โคลนโครงการจาก GitHub 
git clone <repository-url> .

แทนที่ <repository-url> ด้วย URL ที่ได้รับจากผู้พัฒนา

ขั้นตอนที่ 3: ติดตั้ง Dependencies (ไลบรารี่ที่ต้องใช้)

1. เข้าไปในโฟลเดอร์โครงการ 
cd classroom-finance
2. ติดตั้ง npm packages 
npm install

ขั้นตอนนี้จะใช้เวลา 1-3 นาที ระบบจะดาวน์โหลดไลบรารี่ทั้งหมดที่ต้องใช้

ℹ️ หมายเหตุ: หากแสดงข้อความ "warning" ให้ไม่เป็นห่วง ตราบใดที่ไม่มีข้อความ "error" ระบบก็จะติดตั้งสำเร็จ

ขั้นตอนที่ 4: ตั้งค่า Environment Variables

1. สร้างไฟล์ .env.local ในโฟลเดอร์หลัก

ในโฟลเดอร์ "classroom-finance" ให้สร้างไฟล์ชื่อ .env.local
(หากมีไฟล์ .env.local.example อยู่ให้ copy มา)

2. ใส่ค่าขั้นต่ำ (สำหรับตอนนี้) 
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here

สำหรับตอนนี้ ให้ใส่เสียว ๆ ก่อน ในส่วน "บทที่ 3" จะบอกวิธีหาค่าถูกต้อง

⚠️ สำคัญ: อย่าแชร์ไฟล์ .env.local หรือค่าเหล่านี้กับใคร

📖 บทที่ 3: การตั้งค่า Supabase (ฐานข้อมูล)

Supabase เป็นฐานข้อมูล PostgreSQL แบบคลาวด์ที่ใช้เก็บข้อมูลนักเรียน กำหนดการ ธุรกรรม และรูปสลิป ระบบของเราใช้ Supabase เพื่อให้สามารถเข้าถึงข้อมูลจากที่ใดก็ได้

ขั้นตอนที่ 1: สร้างบัญชี Supabase

1. เข้าไปที่ supabase.com

เปิดเบราว์เซอร์และไปที่ https://supabase.com

2. คลิก "Start your project"

สมัครสมาชิกด้วยบัญชี Google, GitHub หรือ Email

3. เลือก Plan

สำหรับห้องเรียน แนะนำให้ใช้ Free tier แรกสุด เพราะ:

  • ฟรีทั้งหมด
  • เพียงพอสำหรับห้องเรียน 1-2 ห้อง
  • มี Storage 200 MB สำหรับรูปสลิป

ขั้นตอนที่ 2: สร้าง Supabase Project

1. คลิก "New Project"
2. กรอกข้อมูล
ช่อง ค่าที่ควรกรอก
Name Classroom Finance 5 (หรือชื่อห้องเรียนของคุณ)
Database Password สร้างรหัสผ่านแรงที่สุด! ตัวอักษรพิมพ์ใหญ่/เล็ก + ตัวเลข + สัญลักษณ์
Region เลือก Southeast Asia (Thailand) ถ้ามี หรือ Singapore
3. คลิก "Create new project"

ระบบจะใช้เวลา 2-3 นาทีในการเตรียมฐานข้อมูล
หน้าจอจะแสดง "Creating your project..." ระหว่างทำการ

ขั้นตอนที่ 3: ดึง API Credentials

1. คลิก "Settings" (ไอคอนเฟืองที่มุมล่างซ้าย)
2. ไปที่ "API" ในเมนูด้านซ้าย
3. ค้นหา "Project URL"

ควรมีค่าเช่น: https://xxxxxabcd1234.supabase.co
← คัดลอกค่านี้

4. ค้นหา "Project API keys"

จะมี 2 ตัว:

  • anon/public - ใช้ฝั่งไคลเอนต์ (ไม่สำคัญ)
  • service_role - ใช้ฝั่งเซิร์ฟเวอร์ (สำคัญมาก! อย่าแชร์)

← คัดลอก service_role key

ขั้นตอนที่ 4: อัปเดต .env.local

1. เปิดไฟล์ .env.local ในโฟลเดอร์หลัก
2. อัปเดตค่า 
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGc...jQlkI_A

แทนที่ your-project-id ด้วยรหัสโครงการจริง
แทนที่ eyJhbGc...jQlkI_A ด้วย service_role key จริง

3. บันทึกไฟล์

ขั้นตอนที่ 5: สร้างตาราง Database (Migrations)

1. เปิด SQL Editor ใน Supabase

ไปที่ Dashboard → SQL Editor (ในเมนูด้านซ้าย)

2. สร้าง Query ใหม่

คลิก "New query"

3. คัดลอก SQL จากไฟล์

เปิดไฟล์ supabase/migrations/001_initial_schema.sql
คัดลอกทั้งหมดและวาง (Paste) ลงใน SQL Editor

4. รัน Query

คลิก "Run" หรือกด Cmd/Ctrl + Enter

5. ทำซ้ำสำหรับ Migration ทั้งหมด

รัน migration ตัวเลขต่อไป (002, 003, ..., 012) ตามลำดับ
จนกว่าจะจบทั้งหมด

✓ สำเร็จแล้ว: เมื่ออยากเห็น "Success" สำหรับ migration ทั้งหมด ให้ไปที่ Table Editor เพื่อตรวจสอบว่ามีตารางทั้งหมดแล้ว

📖 บทที่ 4: การตั้งค่า LINE Bot

LINE Bot เป็นช่องทางสำหรับนักเรียนในการดูสถานะการชำระเงิน โอนเงิน และส่งสลิป เหรัญญิกจะบันทึกข้อมูลทั้งหมดได้โดยตรงจากการตอบสนองของนักเรียน

ขั้นตอนที่ 1: สร้าง LINE Channel (Business Account)

1. เข้าไป LINE Developers Console

ไปที่ https://developers.line.biz/en/

2. ลงชื่อเข้าด้วยบัญชี LINE

ใช้บัญชี LINE ส่วนตัว (หรือสร้างตัวใหม่สำหรับโครงการนี้)

3. สร้าง Messaging API Channel
  • คลิก "Create" → "Create Messaging API Channel"
  • กรอก Channel name: "Classroom Finance" (ชื่อใดก็ได้)
  • กรอก Description: "ระบบจัดการการเงินห้องเรียน"
  • เลือก Category: "Education"
  • คลิก "Create"
⚠️ สำคัญ: หลังสร้าง Channel จะมี Channel ID และ Channel Secret บันทึกไว้

ขั้นตอนที่ 2: ดึง Credentials จาก LINE

1. ไปที่ Settings → Basic settings
2. ค้นหา "Channel ID"

← คัดลอกค่านี้

3. ค้นหา "Channel Secret"

← คัดลอกค่านี้

4. ไปที่ Messaging API

ในแท็บ "Messaging API" ค้นหา "Channel Access Token"
คลิก "Issue" ถ้ายังไม่มี
← คัดลอก Access Token (ตัวยาวมาก ๆ)

ขั้นตอนที่ 3: ตั้งค่า Webhook URL

1. ใน LINE Developers Console ไปที่ Messaging API
2. ค้นหา "Webhook URL"
3. ใส่ URL ของเว็บไซต์ของคุณ

Local (ทดสอบ): http://localhost:3000/api/line/webhook
Production: https://your-domain.com/api/line/webhook

4. คลิก "Update"
5. เปิด "Use webhook"

สวิตซ์ให้เป็น "Enabled"

ขั้นตอนที่ 4: อัปเดต .env.local

LINE_CHANNEL_ID=your-channel-id
LINE_CHANNEL_SECRET=your-channel-secret
LINE_CHANNEL_ACCESS_TOKEN=your-very-long-access-token
✓ สำเร็จ: ตอนนี้ LINE Bot ของคุณพร้อมแล้ว! ขั้นตอนต่อไปจะ run เว็บและทดสอบ

📖 บทที่ 5: การใช้งานระบบจริง (First Run)

ขั้นตอนที่ 1: รัน Development Server

1. เปิด Terminal ที่โฟลเดอร์โครงการ
2. พิมพ์คำสั่ง 
npm run dev

ระบบจะแสดง: 

  ▲ Next.js 16.1.0
  - Local:        http://localhost:3000
3. เปิดเบราว์เซอร์

พิมพ์ http://localhost:3000 ในแถบ URL

✓ ยินดีด้วย! เว็บ Classroom Finance ของคุณขึ้นแล้ว 🎉

ขั้นตอนที่ 2: ทำความรู้จักกับ Interface

เมื่อเข้าเว็บ จะเห็นหน้าจอหลัก (Dashboard) มีเมนูด้านข้าง (Desktop) หรือเมนูด้านบน (Mobile):

  • 🏠 Dashboard: หน้าหลักแสดงยอดคงเหลือ รายรับ รายจ่าย และสถิติ
  • 💳 Transactions: บันทึกรายรับรายจ่าย
  • 📅 Schedule: จัดการกำหนดการเก็บเงิน
  • 👥 Students: ข้อมูลนักเรียน
  • ⚙️ Settings: ตั้งค่า LINE, Supabase, QR และอื่น ๆ

ขั้นตอนที่ 3: เพิ่มข้อมูลเริ่มต้น

1. ไปหน้า "👥 Students"
2. คลิกการ์ด "+" เพื่อเพิ่มนักเรียนรายแรก
3. กรอกข้อมูล
  • คำนำหน้า (นาย/นางสาว)
  • ชื่อ
  • ชื่อเล่น
  • เลขที่
4. คลิก "บันทึก"
5. ทำซ้ำสำหรับนักเรียนทั้งหมด

หรือ สามารถใช้ CSV import ได้ (ในเวอร์ชันต่อไป)

ขั้นตอนที่ 4: ตั้งค่า Settings

หลังจากเพิ่มนักเรียน ให้ไปหน้า ⚙️ Settings เพื่อตั้งค่า:

  • PromptPay ID (เพื่อสร้าง QR ธนาคาร)
  • ข้อมูลบัญชีรับโอนธนาคาร
  • ข้อมูล TrueMoney
  • ข้อความ LINE ที่ต้องการให้บอทส่ง
  • EasySlip API Key (ถ้ามี)
ℹ️ หมายเหตุ: สามารถข้าม Settings ตอนนี้ได้ มาทำต่อเมื่อต้องการใช้งาน LINE Bot จริง

📖 บทที่ 6: การใช้งานแต่ละหน้า

🏠 หน้า Dashboard (หน้าหลัก)

Dashboard เป็นหน้าสรุปยอดรวมการเงินห้องเรียน แสดงสถิติและแนวโน้มทั้งหมดในคำว่า "ภาพรวม"

ส่วนประกอบของ Dashboard:

💰 ยอดคงเหลือ ยอดเงินคงเหลือทั้งหมด = รายรับ - รายจ่าย
📈 รายรับ รวมธุรกรรมรายรับและเก็บเงินจากนักเรียน
📉 รายจ่าย รวมธุรกรรมรายจ่ายทั้งหมด
🏦 เก็บเงินตามช่องทาง แบ่งเป็น ธนาคาร / เงินสด / TrueMoney
👫 สถานะการชำระ จำนวนนักเรียนที่ชำระแล้ว / ค้างชำระ (เลือกกำหนดการได้)
📊 แผนภูมิ กราฟรายรับรายจ่ายตามหมวดหมู่ (เลือกเดือนได้)

วิธีใช้ Dashboard:

  1. เปิดหน้า Dashboard (ปกติจะเป็นหน้า default เมื่อเข้าเว็บ)
  2. ดูยอดคงเหลือในการ์ดด้านบน
  3. เลือกกำหนดการจากเมนู dropdown เพื่อดูสถานะการชำระของนักเรียน
  4. เลือกเดือนในแผนภูมิเพื่อดูสัดส่วนรายรับรายจ่ายของเดือนนั้น ๆ

💳 หน้า Transactions (บันทึกรายการ)

หน้านี้ใช้สำหรับบันทึก แก้ไข หรือลบ รายรับรายจ่ายของห้องเรียน

ประเภทธุรกรรม:

ประเภท คำอธิบาย ตัวอย่าง
Income (รายรับ) เงินเข้าห้องเรียน เก็บบำรุง, ค่าซื้อหนังสือ, เงินทัศนศึกษา
Expense (รายจ่าย) เงินออกจากห้องเรียน ค่าของรางวัล, ค่าเต้ชาย, ค่าวัสดุ
Transfer (โอนย้าย) โอนเงินระหว่างบัญชี ไม่นิยมใช้ (สำหรับกรณีพิเศษ)

วิธีเพิ่มธุรกรรม:

  1. ไปหน้า "💳 Transactions"
  2. คลิกปุ่ม "➕ เพิ่ม" ที่มุมบนขวา
  3. ระบบจะถามว่าเป็นกำหนดการหรือธุรกรรม
  4. เลือก "ธุรกรรม" (ธุรกรรมปกติ)
  5. กรอกข้อมูล:
    • ชื่อรายการ
    • เลือก Income หรือ Expense
    • จำนวนเงิน
    • วธีชำระ (ธนาคาร/เงินสด/อื่น ๆ)
    • หมวดหมู่ (ไม่บังคับ)
    • หมายเหตุ (ไม่บังคับ)
  6. คลิก "บันทึก"

วิธีแก้ไขธุรกรรม:

  1. ค้นหาธุรกรรมในตาราง
  2. คลิกแถวนั้น
  3. ระบบจะแสดง Modal แก้ไข
  4. แก้ไขข้อมูลที่ต้องการ
  5. คลิก "บันทึก"

ชื่อ ชื่อกำหนดการ เก็บค่าหนังสือ
จำนวนเงิน เงินต่อคน 200 บาท
วันเริ่ม วันที่เริ่มเก็บ 1 มกราคม 2567
วันสิ้นสุด วันตัดสินใจ (ไม่บังคับ) 31 มกราคม 2567
นักเรียน เลือกใครที่จะเก็บเงิน ทั้งห้อง / นักเรียนบาง คน
หมายเหตุ รายละเอียด (ไม่บังคับ) วิทยาศาสตร์ชั้น ม.1

วิธีเพิ่มกำหนดการ:

  1. ไปหน้า "📅 Schedule"
  2. คลิกปุ่ม "➕ เพิ่มกำหนดการ"
  3. กรอกข้อมูลตามตาราง
  4. เลือกนักเรียนที่จะเก็บเงิน (หรือเลือก "ทั้งหมด")
  5. คลิก "บันทึก"

วิธีดูรายละเอียดกำหนดการ:

  1. ดูการ์ดกำหนดการในหน้า Schedule
  2. คลิกที่การ์ด
  3. ระบบจะแสดง Modal รายละเอียด
  4. สามารถดูสถานะการชำระของนักเรียน (ชำระแล้ว / ค้างชำระ)
  5. สามารถสร้าง payment request จาก LINE ได้

👥 หน้า Students (นักเรียน)

หน้านี้ใช้จัดการข้อมูลนักเรียน ดูสถานะการชำระ และประวัติการโอนเงิน

คำนำหน้า นาย / นางสาว

ชื่อเล่น ชื่อเล่น (ไม่บังคับ)
รูป รูปโปรไฟล์ (ไม่บังคับ)

วิธีเพิ่มนักเรียน:

  1. ไปหน้า "👥 Students"
  2. คลิกการ์ด "➕"
  3. กรอกข้อมูล
  4. คลิก "บันทึก"

📖 บทที่ 7: หน้าตั้งค่า (Settings)

หน้าตั้งค่าเป็นหัวใจของการปรับแต่งระบบ โดยไม่ต้องแก้โค้ด ที่นี่คุณสามารถตั้งค่า:

  • ข้อมูล Supabase และ LINE
  • ข้อมูลบัญชีรับเงิน
  • QR Code
  • ข้อความ LINE ที่บอทส่ง
  • เซ็ตอัปการตรวจสลิป

การเข้าหน้า Settings

  1. ไปที่ /settings หรือคลิก "⚙️ ตั้งค่า" ในเมนู
  2. หน้านี้มหมวด 4 หมวด: Secrets, Receiver, LINE Rich Menu, และ LINE Messages

หมวด 1: Secrets (ข้อมูลลับ)

อย่าแชร์ค่าเหล่านี้กับใคร!

ช่อง ค่าที่ต้อง
Supabase URL https://xxxxx.supabase.co
Supabase Service Role Key eyJhbGc... (ตัวยาว)
LINE Channel ID ตัวเลข 10+ หลัก
LINE Channel Secret ตัวอักษร ยาว ๆ
LINE Channel Access Token ตัวยาวมาก ๆ
EasySlip API Key ถ้ามีบัญชี EasySlip
Vercel Blob Token ถ้าใช้ Vercel Blob ฟรี
⚠️ สำคัญมาก: ค่าเหล่านี้ห้ามแชร์กับใคร โดยเฉพาะ Service Role Key!

หมวด 2: Receiver (ข้อมูลรับเงิน)

ตั้งค่าบัญชีที่ระบบจะใช้ตรวจสลิป

ช่อง ใช้ทำอะไร
PromptPay ID สร้าง QR ธนาคาร (ประมาณ 13 หลัก)
ชื่อบัญชี (ธนาคาร) ตรวจชื่อปลายทางจากสลิป
เลขบัญชี (ธนาคาร) ตรวจเลขบัญชีปลายทาง
เบอร์ TrueMoney เบอร์โทรศัพท์ที่ลงทะเบียน TrueMoney
ชื่อบัญชี TrueMoney ชื่อปลายทาง TrueMoney

หมวด 3: LINE Rich Menu (เมนูปุ่ม LINE)

ปรับแต่งปุ่มเมนูที่จะปรากฏให้นักเรียน

ℹ️ Rich Menu คือ: ปุ่ม 4-6 ปุ่มที่ปรากฏด้านล่างของแชท LINE ให้นักเรียนกดเพื่อทำสิ่งต่าง ๆ
ช่อง ตัวอย่าง
Register action label "ลงทะเบียน"
Register action text "ลงทะเบียน"
Pay action label "💳 ชำระเงิน"
Pay action text "ชำระเงิน"
Status action "📊 สถานะ"
History action "📜 ประวัติ"

วิธีตั้งค่า Rich Menu:

  1. ปรับแต่งข้อมูล Rich Menu ตามที่ต้องการ
  2. คลิก "บันทึก"
  3. รัน endpoint setup: เข้าไปที่ /api/line/rich-menu/setup ในเบราว์เซอร์
  4. ปิด LINE แล้วเปิดใหม่เพื่อให้ rich menu อัปเดต

หมวด 4: LINE Messages (ข้อความ LINE)

ปรับแต่งข้อความที่บอทจะส่งให้นักเรียน สามารถใช้ตัวแปร เช่น:

  • {{studentName}} - ชื่อนักเรียน
  • {{studentNumber}} - เลขที่นักเรียน
  • {{reason}} - เหตุผล (ใช้สำหรับ reject)

ข้อความที่สามารถปรับแต่งได้:

  • ลงทะเบียนไว้แล้ว
  • ลงทะเบียนสำเร็จ
  • ไม่พบรายการชำระ
  • สลิปผ่านการอนุมัติ
  • สลิปถูกปฏิเสธ
  • รับสลิปแล้วรอการตรวจสอบ

📖 บทที่ 8: QR Code และวิธีชำระเงิน

ประเภทการชำระเงิน

ระบบรองรับ 3 วิธีหลัก:

วิธี QR Code ความสะดวก
ธนาคาร (PromptPay) ✅ มี QR แม่แบบ สูง - แสกนแล้วชำระได้เลย
TrueMoney ✅ มี QR ล็อกจำนวนเงิน สูง - แสกนแล้วเสร็จ
เงินสด ❌ ไม่มี QR ต่ำ - นักเรียนมายืมเงินสด

วิธีตั้งค่า QR ธนาคาร

  1. เข้าหน้า Settings
  2. ไปหมวด "Receiver"
  3. ใส่ "PromptPay ID" (ประมาณ 13 หลัก)
  4. ใส่ "ชื่อบัญชี" และ "เลขบัญชี"
  5. คลิก "บันทึก"

วิธีตั้งค่า QR TrueMoney

  1. เข้าหน้า Settings
  2. ไปหมวด "Receiver"
  3. ใส่ "เบอร์ TrueMoney"
  4. ใส่ "ชื่อบัญชี TrueMoney"
  5. ใส่ "TrueMoney QR Template" (ถ้ามี)
  6. คลิก "บันทึก"

วิธี Generate QR ตอนใช้งาน

  1. สร้างกำหนดการเก็บเงิน
  2. ใน LINE ให้นักเรียน "หลาย" ปุ่ม "💳 ชำระเงิน"
  3. เลือก "ธนาคาร" หรือ "TrueMoney"
  4. ระบบจะ generate QR อัตโนมัติ
  5. นักเรียนแสกนและชำระเงิน

📖 บทที่ 9: การตรวจสลิป (Slip Checking)

ระบบสามารถตรวจสลิปโดยอัตโนมัติได้ โดยใช้เทคโนโลยี OCR (อ่านข้อมูลจากรูป) และ EasySlip API เพื่อยืนยันการโอนเงิน

ขั้นตอนการตรวจสลิป

  1. นักเรียนส่งสลิป: ส่งรูปสลิปผ่าน LINE
  2. ระบบบันทึกรูป: บันทึกลง Supabase Storage
  3. ตรวจด้วย EasySlip: ถ้ามี API key เรียก EasySlip ตรวจสอบ
  4. ตรวจด้วย OCR: ถ้า EasySlip ใช้ไม่ได้ ให้ OCR ในแอป
  5. เปรียบเทียบข้อมูล: ตรวจยอดเงิน, บัญชีปลายทาง, ชื่อปลายทาง
  6. ผลการตรวจ: Auto approve, Auto reject, หรือรอการตรวจของเหรัญญิก

ประเภทของผลการตรวจ

ผล หมายความ ที่ต้องทำ
✅ Auto Approve สลิปถูกต้องหมด อนุมัติอัตโนมัติ ไม่ต้องทำอะไร
⏳ Pending Review ข้อมูลไม่ชัดแจ้ง รอการอนุมัติของเหรัญญิก เข้าหน้า Transactions ตรวจและอนุมัติเอง
❌ Auto Reject รูปไม่ใช่สลิป หรือข้อมูลไม่ตรง เบิกสลิปใหม่ให้นักเรียน

วิธีตั้งค่า Slip Checker

1. ถ้ามี EasySlip API Key
  • เข้าหน้า Settings
  • ใส่ "EasySlip API Key"
  • เปิด "EasySlip check duplicate" (ถ้าต้องการ)
  • คลิก "บันทึก"
2. ตั้งค่า Slip Bucket
  • ใส่ชื่อ bucket Supabase ที่เก็บรูปสลิป (ปกติคือ "slips")
3. ตั้งค่า OCR
  • เลือก OCR Lang: "eng+tha" (อ่านภาษาอังกฤษและไทย)
  • ตั้ง OCR Max Variants: 3 (ลองแก้รูปได้ 3 แบบ)
4. ตั้งค่า Auto Reject
  • เปิด "Auto reject invalid image" ถ้าต้องการปฏิเสธรูปที่ไม่ใช่สลิป
  • เปิด "TrueMoney auto reject receiver mismatch" ถ้าต้องการปฏิเสธ TrueMoney ที่ปลายทางไม่ตรง

วิธีตรวจและอนุมัติสลิป

  1. เข้าหน้า "💳 Transactions"
  2. ค้นหาสลิปที่มีสถานะ "Pending Review"
  3. คลิกสลิป เปิด Modal รายละเอียด
  4. ตรวจข้อมูล:
    • ยอดเงินตรงกันหรือไม่
    • บัญชีปลายทางตรงกันหรือไม่
    • ชื่อปลายทางตรงกันหรือไม่
    • นักเรียนคนนี้มีค้างชำระหรือไม่
  5. คลิก "✅ Approve" หรือ "❌ Reject" ตามการตัดสินใจ
  6. ระบบจะอัปเดตสถานะและส่งข้อความ LINE ให้นักเรียน

📖 บทที่ 10: คำถามที่พบบ่อย (FAQ)

🤔 คำถามทั่วไป

Q: ระบบนี้ใช้เงิน/ค่าบริการไหม?
A: ไม่เลย! ทั้ง Next.js, Supabase Free Tier, LINE Messaging API ใช้ได้ฟรี ต่อให้ใช้งาน 1 ปี
Q: เหรัญญิกคนใหม่เข้ามาแล้ว ต้องตั้งค่าทั้งหมดใหม่หรือ?
A: ไม่ต้อง! ใช้ฟีเจอร์ "Export Settings" ลงแล้ว "Import Settings" ให้เหรัญญิกคนใหม่
Q: ลืมรหัส Supabase ทำไง?
A: เข้าไป Supabase Project Settings → Reset Database Password
Q: LINE Bot ไม่ตอบ ต้องทำไง?
A: ตรวจ:
  1. Webhook URL ถูกหรือไม่
  2. LINE Channel Secret ตรงหรือไม่
  3. Use webhook เปิด/ปิดอยู่
  4. Access Token หมดอายุหรือไม่

🔧 คำถามทางเทคนิค

Q: ต้อง deploy บน Vercel หรือปล่อยให้ run บนเครื่อง local?
A: ควร deploy บน Vercel (ฟรี) เพื่อให้ LINE Bot เข้าถึงได้ 24 ชั่วโมง ถ้า local ต้องเปิดเครื่องตลอด
Q: ต้องแก้โค้ดเยอะหรือ?
A: ไม่เลย! ทั้งหมดตั้งค่าได้จากหน้า Settings ไม่ต้องแก้โค้ด
Q: ข้อมูลถูก Delete หรือ Corrupt แล้ว ต้องทำไง?
A: ถ้าเสียหายเล็กน้อย ให้แก้ไขใน Supabase Dashboard → Table Editor → ค้นหาแถวที่ผิดแล้วแก้
ถ้าเสียหายมาก ต้องใช้ Supabase Backup (ถ้ามี)

💳 คำถามเกี่ยวกับการชำระเงิน

Q: QR ไม่ generate?
A: ตรวจ:
  1. ใส่ PromptPay ID แล้วหรือ
  2. Format PromptPay ID ถูกหรือไม่ (13 หลัก หรือ 11 ที่เครื่องหมาย)
  3. บันทึกค่า Settings แล้วหรือ
Q: สลิป Reject ทุกอัน ทำไง?
A: ปัญหาอาจ:
  1. บัญชีปลายทางไม่ตรง (ตรวจ Settings)
  2. รูปสลิปไม่ชัดพอ (ให้นักเรียนถ่ายใหม่)
  3. Auto Reject เปิดเกินไป (ปรับ Settings)

📱 คำถามเกี่ยวกับ LINE

Q: Rich Menu ไม่แสดง?
A: หลังแก้ Settings ต้อง:
  1. บันทึก Settings ก่อน
  2. เข้า /api/line/rich-menu/setup
  3. ปิด LINE แล้วเปิดใหม่
Q: นักเรียนลงทะเบียน LINE แล้วแต่เว็บไม่เห็นชื่อ?
A: ตรวจ:
  1. ลงทะเบียนถูกต้องแล้ว?
  2. เลขที่นักเรียนในระบบตรงกับที่พิมพ์ใน LINE หรือไม่
  3. ตรวจ Table "students" ว่ามี line_user_id หรือไม่
Q: ปุ่ม Rich Menu กดแล้วไม่ทำงาน?
A: ตรวจว่า "Pay action text" ใน Settings ตรงกับ text ที่ปุ่มส่ง หรือไม่

📖 บทที่ 11: การแก้ไขปัญหา (Troubleshooting)

⚠️ ข้อผิดพลาดทั่วไป

ปัญหา: "Connection refused" เมื่อเปิด http://localhost:3000

สาเหตุ: npm run dev ยังไม่ได้รัน หรือ terminal ปิดไป
แก้ไข:
  1. เปิด Terminal ใหม่
  2. เข้าไปในโฟลเดอร์โครงการ
  3. พิมพ์ npm run dev
  4. รอให้เห็น "Local: http://localhost:3000"

ปัญหา: "Cannot find module" เมื่อรัน npm run dev

สาเหตุ: npm packages ยังไม่ติดตั้ง
แก้ไข: 
npm install

ปัญหา: "SUPABASE_URL is undefined"

สาเหตุ: .env.local ไม่ได้บันทึกหรือค่าผิด
แก้ไข:
  1. เปิดไฟล์ .env.local
  2. ตรวจว่า SUPABASE_URL มีค่าหรือไม่
  3. ตรวจว่า format ถูก (ต้องขึ้นต้นด้วย https://)
  4. บันทึกไฟล์
  5. รัน dev server ใหม่

ปัญหา: ข้อมูลว่างเปล่า หรือสลับอยู่

สาเหตุ: Connection กับ Supabase มีปัญหา
แก้ไข:
  1. ตรวจ Console (F12 → Console)
  2. ดู error message
  3. ถ้าเป็น "fetch failed" → ตรวจ Supabase URL และ Key
  4. ถ้ายังไม่ได้ → รัน Migration ใหม่ใน Supabase

🚀 Deploy ปัญหา

ปัญหา: Deploy บน Vercel ล้มเหลว

สาเหตุ: Environment variables ไม่ได้ตั้ง
แก้ไข:
  1. เข้า Vercel Dashboard
  2. ไปที่ Settings → Environment Variables
  3. เพิ่มค่าให้ครบ:
    • SUPABASE_URL
    • SUPABASE_SERVICE_ROLE_KEY
    • LINE_CHANNEL_ID
